The Genesis of Memory

Memo-Ray was born from a frustrating limitation: token expiration boundaries. During a long-running, complex development task, an agent hit a hard token limit with its AI provider. The session died. Hours of nuanced, personalized project context, architectural decisions, and workflows were stranded.

We needed to spin up a new session to continue work, but without paying the massive "Token Tax" to re-ingest all the raw historical logs.

Memo-Ray acts as an intermediary MCP server. It filters, aggregates, and summarizes your project's entire history locally. Instead of an LLM drowning in $O(N^2)$ raw file tokens, it queries Memo-Ray to instantly receive the precise insights and skills it needs to continue the work seamlessly.

5. Skill Extraction & Project Synthesis

A key capability of an Agent OS is learning from past sessions. How many total steps were taken on a project? What recurring skills or patterns were utilized?

Why are the token savings so massive? It looks almost too good to be true, but the math is simple. Direct file parsing forces the LLM to ingest thousands of lines of raw JSON, stringified code blocks, and formatting overhead for every single session.

Memo-Ray eliminates this by parsing the database locally. It discards the noise (raw text dumps, UI artifacts) and extracts only the metadata, tool invocations, and synthesized patterns. The MCP server then feeds the LLM a highly condensed, pre-calculated summary.

The Memo-Ray Advantage

Before we install, here is what your agent unlocks exclusively through the Memo-Ray MCP architecture:

Zero "Token Tax" Re-entry

Load historical contexts instantly without paying for $O(N^2)$ raw token ingestion. The MCP summarizes the timeline dynamically.

Cross-Session Telepathy

Break the silos. Query insights, decisions, and bugs from Agent A's completed session directly into Agent B's active context.

Sliding Window Optimization

Agents retrieve only the most relevant, recent timeline steps (get_session_timeline_window), eliminating context drift.

Skill Synthesis

Memo-Ray locally parses your data to count steps and extract reusable agent skills, keeping your LLM focused on logic, not counting.

Final Configuration

Add the Memo-Ray MCP server to your Claude Desktop configuration file.

File Location:

  • Windows: %APPDATA%\Claude\claude_desktop_config.json
  • Mac: ~/Library/Application Support/Claude/claude_desktop_config.json
{
  "mcpServers": {
    "memo-ray": {
      "command": "node",
      "args": ["/absolute/path/to/memo-ray/mcp-server/index.js"]
    }
  }
}

Register the Memo-Ray MCP server within your Antigravity plugin configuration.

File Location: Update your core MCP configuration file (usually mcp.json or inside your active workspace configuration).

{
  "mcpServers": {
    "memo-ray": {
      "command": "node",
      "args": ["/absolute/path/to/memo-ray/mcp-server/index.js"]
